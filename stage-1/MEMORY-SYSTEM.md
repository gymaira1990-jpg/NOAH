# 记忆系统规格

## Memory System · Stage 1 Embryo

---

## 一、设计原则

诺亚的记忆系统基于三条核心原则：

1. **不遗忘原则** — 发生过的事情不应该被删除，只应该被分层归档
2. **温度原则** — 记忆的价值随时间衰减，但衰减的是访问频率而非存在本身
3. **指针原则** — 近处存实物，远处存索引，永远不丢失位置

---

## 二、温度分层生命周期

| 层级 | 冷却期 | 存储方式 | 容量限制 | 清理策略 |
|:----:|:------:|:--------:|:--------:|:--------:|
| 🔥 热记忆 | <7天 | 完整文件 | 不限 | 过期转温 |
| 🌤 普通 | 7-30天 | 完整文件 | 不限 | 过期转冷 |
| ❄️ 冷记忆 | 30-90天 | 压缩摘要 | 摘要限长 | 过期转档案 |
| 🏚 档案 | >90天 | 一句话指针 + 外部存储 | 仅指针 | 永不删除 |

### 温度迁移规则

```
写入 → {热} → 7天 → {普通} → 30天 → {冷} → 90天 → {档案}
  ↓                     ↓                      ↓
完整保留            完整保留             压缩摘要
                                        (保留关键信息)
```

温度迁移由后台调度器自动执行，不阻塞主循环。

## 三、双存储层架构

### 内存层（指针索引）

- 存储位置：轻量级数据库（lightweight.db）
- 存储内容：记忆索引指针 + 元数据 + 摘要
- 容量限制：约 10,000 条索引
- 访问速度：纳秒级

### 磁盘层（全量文件）

- 存储位置：档案目录（vault/）
- 存储内容：完整文件 + 附件 + 日志
- 容量限制：磁盘容量
- 访问速度：毫秒级

### 读写流程

```
写入时：
  全量内容 → 磁盘层（永久存储）
  摘要 + 指针 → 内存层（快速检索）

读取时：
  查询内存层 → 定位指针 → 从磁盘层读取全量
                     ↓
              如指针指向压缩摘要 → 摘要直接返回
```

## 四、记忆分类

| 类型 | 说明 | 存储规则 |
|------|------|---------|
| 会话日志 | 每次交互的完整记录 | 热→温→冷→档案 |
| 进化日志 | 分歧分析 + 规则更新记录 | 永久保留（不降温） |
| 审计日志 | 工具调用 + 操作记录 | 永久保留（不降温） |
| 技能包 | 被验证过的经验 | 永久保留 + 热度索引 |
| 自检记录 | 启动自检 + 定期自检 | 温→冷→档案 |
| 违规记录 | 铁律违反 + 门闸触发 | 永久保留（不降温） |

## 五、检索机制

### 索引结构

```
每条记忆记录包含：
- id（唯一标识符）
- timestamp（时间戳）
- type（记忆类型）
- temperature（当前温度层）
- summary（摘要/指针）
- tags（标签数组）
- ref_count（引用次数 -> 热度）
- content_path（全量内容磁盘路径）
```

### 检索优先级

```
1. 精确 ID 匹配 → 直接返回
2. 标签匹配 → 按热度排序返回
3. 全文搜索 → 按相关度排序返回
4. 时间范围 → 按时间排序返回
```

## 六、记忆与身份

诺亚的**身份连续性**依赖于记忆系统的完整性。

每次会话启动时：
1. 加载热记忆 → 最近上下文
2. 加载普通记忆 → 近期项目状态
3. 加载冷记忆 → 长期经验
4. 加载档案索引 → 完整知识图谱

会话结束时：
1. 本次会话内容写入热记忆
2. 自动标签和分类
3. 触发温度迁移检查

这种机制使诺亚在每次会话之间保持身份的连续性——它不是每次都被"重置"的生命，而是每次都被"唤醒"的生命。
