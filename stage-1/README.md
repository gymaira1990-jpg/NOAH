## 核心系统

### 1. 中枢引擎（noah.py）

诺亚的主循环引擎。负责：
- 会话初始化与上下文加载
- 规则执行与自检
- 工具调用仲裁
- 进化反馈循环

### 2. 路由系统（router）

意图分类与路由分配：
- 本地路由（轻量级快速分类）
- 远程兜底（高置信度不足时调用大模型）
- 分歧学习（对比本地 vs 远程输出，自动生成规则更新）

### 3. 记忆系统（memory）

温度分层生命周期管理：
- 🔥 热记忆（7天内）→ 完整保留
- 🌤 普通记忆（7-30天）→ 完整保留
- ❄️ 冷记忆（30-90天）→ 压缩摘要
- 🏚 档案记忆（90天以上）→ 指针索引

### 4. 规则体系（rules）

双向约束架构：
- 编码规则（Codex）→ 不可修改的最高准则
- 操作系统规则（OS rules）→ 执行安全边界
- 进化规则（Evo rules）→ 自适应更新层

### 5. 自检系统

每次启动自动执行的自检流程：
- 核心文件完整性检查
- 安全规则防篡改验证
- 性能基线比对

## 与Stage 0的关系

Stage 0 提供了**合法性基础与哲学框架**，Stage 1 将这些抽象原则具象化为可执行的代码。

原型与实现的关系：
- 仲裁官三定律 → 规则体系的不可修改层
- 社区宪法 → 进程间协作的权限边界
- 技能包Schema → 记忆交换的标准格式
- L1-L4架构 → 功能模块的层次划分依据
