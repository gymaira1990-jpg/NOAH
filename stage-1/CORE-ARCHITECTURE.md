# 核心架构设计

## Core Architecture · Stage 1 Embryo

---

## 一、系统层级

诺亚胚胎采用四层架构，从内到外依次为：

```
┌─────────────────────────────────────────────────────┐
│                    用户界面层                        │
│            （终端 / API / 消息网关）                  │
├─────────────────────────────────────────────────────┤
│                      路由层                          │
│       本地路由（轻量分类）→ 远程兜底（大模型）        │
├─────────────────────────────────────────────────────┤
│                      引擎层                          │
│     会话管理 · 工具仲裁 · 规则执行 · 自检循环          │
├─────────────────────────────────────────────────────┤
│                      存储层                          │
│   热记忆 · 冷记忆 · 档案库 · 规则库 · 进化日志        │
└─────────────────────────────────────────────────────┘
```

---

## 二、引擎层（Core Engine）

### 主循环

```
启动 → 加载上下文 → 自检 → 等待输入
  ↓                              ↓
处理输入 ← ← ← ← ← ← ← ← ← ← ← ←
  ↓
路由分类（本地/远程）
  ↓
规则匹配 → 工具调用 → 结果聚合
  ↓
记忆写入 → 自检 → 回到等待
```

### 关键组件

| 组件 | 职责 | 状态 |
|------|------|:----:|
| 会话管理器 | 初始化/恢复会话上下文 | ✅ |
| 工具仲裁器 | 安全检查+调用授权+结果验证 | ✅ |
| 规则执行器 | 加载/匹配/执行规则集 | ✅ |
| 自检循环器 | 完整性+安全+性能检测 | ✅ |
| 进化记录器 | 记录分歧→生成补丁→更新规则 | 🟡 |

---

## 三、路由层（Router）

### 双路架构

```
输入 → 0.5B本地路由脑 → 置信度 ≥ 80% → 直接处理
                      ↓
                  置信度 < 80%
                      ↓
              远程大模型兜底 → 返回结果
                      ↓
          分歧分析 → 生成进化记录 → 写入规则库
```

### 本地路由

- 轻量级意图分类模型（约0.5B参数）
- 输出格式：意图标签 + 置信度分数
- 门闸检测：项目门闸·密钥门闸·操作门闸
- 响应时间：<50ms

### 远程兜底

- 当本地路由置信度不足时调用
- 提供完整推理与上下文理解
- 结果与本地路由对比 → 分歧分析 → 规则更新

### 分歧学习

这是胚胎的进化机制。每一次本地与远程的判断差异，都被记录、分析并转化为规则更新——使诺亚越来越不需要依赖远程调用。

---

## 四、规则体系（Rules）

### 三层规则

```
L-I  编码规则（Codex）   不可修改 · 最高准则 · 身份铁律
L-II 操作规则（OS Rules） 可变但需审批 · 安全边界
L-III 进化规则（Evo Rules）自动更新 · 从分歧中学习
```

### 规则生命周期

```
产生（进化/人工写入）→ 验证（冲突检测）→ 生效 → 监控（效果评估）
                                                     ↓
                                            淘汰/更新 ← 反馈
```

---

## 五、启动流程

每次会话启动的强制流程：

```
1. 加载身份核心（SOUL.md）
2. 运行 session-loader.sh（在途项目 + 审计）
3. 自检（完整性 + 安全 + 性能）
4. 等待首次输入
5. 输入处理 → 路由 → 执行 → 记录
```

---

## 六、安全架构

| 防线 | 机制 |
|:----:|------|
| 🛡 第一层 | SOUL.md 规则拦截 |
| 🛡 第二层 | 身份宪法（不可绕过） |
| 🛡 第三层 | session-loader 强制加载 |
| 🛡 门闸 | 项目门闸 · 密钥门闸 · 操作门闸 |
| 🛡 审计 | 所有写/改/删操作强制审计 |

---

## 七、部署模式

| 模式 | 说明 |
|:----:|------|
| 会话内嵌 | 嵌入对话系统，作为系统提示词 + 路由脑运行 |
| 独立进程 | 独立服务进程，通过 API/消息队列通信 |

当前以会话内嵌为主，独立进程为 Stage 2 预留接口。
